- 新问题池显著降低了头部平台锚定。

- 累计覆盖约 `0.8506`
- 这说明在“小平台视角”下，系统已经能给出一定程度的组合建议。

### 3. 总排序仍然不是小平台优先

- 原始 `platform_scores` 仍然被 `知乎`、`博客/专栏`、`论坛`、`微信公众号` 等对象主导。
- `baseline_platforms` 与 `niche_opportunities` 在展示层已经分开，但底层总排序依旧偏头部或偏抽象平台桶。
- 当前“小平台优先”更多是展示层和过滤层实现的，不是基础排序已经真正转向小平台。

### 4. 回答正文比系统摘要更好

回答里已经出现很多具体小平台，但很多没有进入最终 `top_actionable_platforms`，例如：

- 雨果网 / 白鲸出海 / InfoQ / MarTech 中国等

这说明：

- 生成层成功逼出了更具体的小平台名单
- 抽取层和平台归一化层没有把这些名字稳定地纳入下游主统计

## 主要问题

### 1. `actionable_platforms` 仍然偏头部

虽然正文里小平台变多，但 `top_actionable_platforms` 仍然是：

- `微信公众号 14`
- `知乎 14`
- `36氪 12`
- `小红书 8`
- `钛媒体 8`

原因不是回答没提到小平台，而是抽取/归类链路对小平台识别不完整。

### 2. 平台归一化覆盖不够

`src/platform_registry.py` 仍缺少大量这次 run 已经出现的平台，例如：

- 站长之家
- A5站长网
- 人人都是产品经理
- 数英网
- 鸟哥笔记
- 运营派
- 掘金
- 博客园
- 本地宝
- 19楼
- 西祠胡同
- 雨果网

导致这些平台虽然在回答正文中出现，却经常停留在 raw text 或 topic claim，没能稳定进入 actionable 结果。

### 3. 证据质量仍然偏弱

- 全量 `60` 条回答中，`59` 条被标为 `weak_evidence`
- `31` 条带有 `generic_listicle` 倾向
- 这说明回答仍然大量依赖概括性行业说法，而不是硬证据支撑

### 4. 结构化稳定性仍有个别失败

- `q018 / web_default` 出现 `preprocess_error`
- 错误为 JSON 解析失败：`Unterminated string...`

虽然失败率不高，但仍说明结构化层在长回答或复杂枚举场景下不够稳。

### 5. 域名提取有回归

`top_domains` 中出现了明显不合法的路径碎片：

- `forum-123-1.html`
- `weight.html`

这说明 `extract_domains()` 仍然会把 URL path 或文件名误判成 domain。

### 6. 仍然存在幻觉风险

很多回答继续给出高精度但证据薄弱的量化说法，例如：

- AI 引用率高 2-3 倍
- 审核通过率 85%
- 收录速度快 40%
- 需求匹配度不足 5% / 可达 60%

这些说法往往只有泛化来源标签、公开资料不足说明，或根本没有足够强的外部可核验证据。

## 当前判断

这次 run 的核心价值在于：

- 证明新问题池方向是对的
- 证明系统现在已经能在回答正文里稳定逼出更具体的小平台

这次 run 暂时还不能证明：

- 最终平台排序已经足够可信
- 小平台名单已经被完整抽取和正确归一
- 量化说法已经足够可靠可直接对外使用
