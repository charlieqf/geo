核心逻辑升级：从“数据抓取”到“认知模拟”
这个Demo的本质是逆向工程（Reverse Engineering）：既然AI是通过学习全网数据来给出答案的，那我们就模拟成千上万个用户去提问，看AI“反复提到了谁”以及“谁的说法最独特”。
第一步：问题生成（Intent Simulation）
不只是生成100个问题，而是生成带有“搜索偏好”的矩阵。
策略： 将关键词通过 Qwen-plus 拆解为四个维度：
直接询问： “某某行业哪家好？”
痛点对比： “A和B选哪个？对比优缺点”
专家建议： “2026年XX行业的发展建议”
长尾避雷： “XX服务有哪些坑？”
Demo亮点： 在页面上展示一个“问题词云”，让客户看到你模拟的问题覆盖了用户所有可能的搜索动机。
第二步：蒸馏（Deep Distillation）
这是最关键的一步，要利用豆包/Qwen的搜索增强功能（RAG/Search API）。
技术细节：
调用豆包API时，必须开启其联网搜索开关。
提取Sources： 豆包返回的结构体中通常带有 references 数组。
数据清洗： 过滤掉百度百科、维基百科等“公海网站”，保留知乎、小红书、独立官网、行业媒体等“可操作网站”。
Demo可视化： 像黑客帝国一样的实时进度条，显示：“正在模拟第45次询问... 捕获到5个新增参考来源”。
第三步：信息熵分析（Independent Contribution）
告诉客户：为什么有的网站发100篇不如有的网站发1篇。
算法逻辑：
如果网站A的内容在所有回答中都被提到，但内容和别人一样，它的熵低。
如果网站B提供了一个独特的论点或数据，且被AI采纳，它的信息量（熵）极高。
数学模型： 使用 Sentence-Transformers 将所有AI给出的原始答案做向量化（Embedding），计算每个网站贡献的文本片段与全集中心的偏离度。偏离度高且被引用多，就是“信息源头”。
可视化效果： 气泡图。气泡大小=引用频率，气泡颜色深浅=独立贡献（熵值）。
第四步：相关性与最小集合（The Golden Set）
把“撒网”变成“狙击”。
相关性分析（Mutual Corroboration）：
如果AI提到“知乎”时经常同时提到“流量玩家官网”，说明两者形成了背书环。
建立关联矩阵：Site A ↔ Site B。
最小覆盖算法（Set Cover Algorithm）：
这是一个经典的优化问题。我们的目标是：用最少的网站，覆盖90%以上的AI提取的事实点。
终极输出（客户最爱）：
推荐名单： “您只需打入这6个网站：知乎(权重35%)、搜狐号(权重15%)、虎嗅(权重12%)...”
ROI预测： “预计部署后，您的品牌在AI搜索中的‘首位建议率’将提升400%。”
技术栈与工具推荐（快速落地版）
前端 UI： Streamlit。
理由： 原生支持 Python，内置了非常漂亮的柱状图、网络图组件，适合做数据分析Demo。
API 网关： One-API 或 Dify。
理由： 方便你同时接入豆包、Qwen和GPT-4，一键切换。
知识图谱可视化： Pyvis。
理由： 能在网页上生成可以拖动的“网站权重关系球”，非常炫酷。
向量处理： ChromaDB（本地轻量级）。
理由： 快速存取AI的答案，计算信息熵。
